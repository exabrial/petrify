# Glossary

**Grove**: The input to Petrify. A DTO holding all ONNX `TreeEnsembleClassifier` parallel arrays. Represents the living trees before petrification.

**Fossil**: The output of Petrify. A generated class implementing `Fossil.predict(float[]) → int`. The decision logic is baked into bytecode — no data structures to navigate at prediction time.

**Node index map**: A `Map<Long, Integer>` built at compile time that translates ONNX logical addresses `(treeId, nodeId)` to physical array indices. ONNX child pointers use logical node IDs scoped per tree; this map resolves them to positions in the flat parallel arrays.

**Logical address**: An ONNX node's identity within its tree — the `(treeId, nodeId)` pair. Child pointers (`nodesTrueNodeIds`, `nodesFalseNodeIds`) reference logical addresses, not array positions.

**Physical address**: The array index where a node's data lives across all parallel arrays (`nodesModes[i]`, `nodesValues[i]`, etc.).

**Class label**: A possible prediction output value. Stored in `classLabelsInt64s`. For binary classification: `{ 0L, 1L }`. For multi-class: `{ 0L, 1L, 2L }` or arbitrary codes. Leaf nodes don't store labels directly — they reference them via `classIds` indirection.

**Class label resolution**: The ONNX indirection chain from a leaf node to its output value. Leaf at `(treeId, nodeId)` → find matching entry in `classNodeIds`/`classTreeIds` → read `classIds[j]` → index into `classLabelsInt64s`. Petrify resolves this at compile time so the generated bytecode contains the literal label value.

**`packLong(int, int)`**: Utility that packs two ints into a single long key: upper 32 bits hold the first int, lower 32 hold the second. Avoids tuple object allocation for map keys.

**Post-transform**: A function applied to accumulated leaf weights after all trees have voted. `NONE` (0) = raw scores, `LOGISTIC` (2) = sigmoid, `SOFTMAX` (1) = softmax across classes. Determines how raw tree outputs become probabilities.

**`BRANCH_LEQ`**: The ONNX split mode meaning "if `features[featureId] <= threshold`, follow the true child; otherwise follow the false child." Represented as byte `1` in `nodesModes`.

**`fcmpg`**: JVM bytecode instruction that compares two floats. Returns `-1` (less), `0` (equal), `1` (greater). The "g" means NaN produces `1`, routing NaN values to the false (right) branch — matching ONNX `BRANCH_LEQ` semantics where `nodesMissingValueTracksTrue` is `0`.

**Operand stack**: The JVM's per-method scratch pad. Bytecode instructions pop inputs from the top, do work, and push results back. Local variable slots (parameters, locals) persist across instructions; the operand stack is transient.

**`ldc`**: JVM bytecode instruction meaning "load constant." Pushes a value from the class file's constant pool onto the operand stack. Used for thresholds, weights, and any literal that doesn't have a dedicated single-byte opcode.
